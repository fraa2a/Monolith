## Current Implementation State

- App/product user-facing branding is now Monolith.
- App window/tray tooltip/mutex/window class use Monolith naming.
- Logs now write to `AppData\Local\Monolith\monolith.log`.
- Replay clips default to `Videos\Monolith\Clips`.
- Manual recordings default to `Videos\Monolith\Recordings`.
- Default config sample paths now match those locations.

## Manual Recording Work

- Added `libs/recording`, a minimal FFmpeg Matroska live writer.
- Added `recording::ManualRecorder` with explicit states:
  - `Idle`
  - `Recording`
  - `Paused`
- `app/recorder/src/main.cpp` now tees encoded H.264/AAC packets to:
  - replay buffer
  - manual recorder when active
- Tray commands now perform real actions:
  - Start Recording
  - Stop Recording
  - Pause Recording
  - Resume Recording
- Hotkeys now match tray labels:
  - `Ctrl+Shift+F8`: save replay
  - `Ctrl+Shift+F9`: start recording
  - `Ctrl+Shift+F10`: stop recording
  - `Ctrl+Shift+F11`: pause/resume recording
- Invalid repeated transitions are rejected and logged without side effects.
- Recording starts on a video keyframe.
- Resume after pause waits for the next video keyframe.
- Paused time is omitted from output timestamps.
